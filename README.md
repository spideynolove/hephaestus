# hephaestus

Automated code improvement loop. Two AI agents alternate — a **Worker** that applies changes and a **Reviewer** that writes targeted feedback — until a fitness score reaches the target or the loop plateaus.

```
┌──────────────────────────────────────────────────────────────────────┐
│  hephaestus/          (lives once, anywhere on your machine)         │
│  ├── orchestrate.sh   loop runner                                    │
│  ├── setup.sh         wizard — points loop at a project              │
│  ├── config.yaml      loop control + agent config                    │
│  ├── .env             active provider + PROJECT_PATH                 │
│  └── logs/                                                           │
│       ├── sequential-thinking/   ← logs for one project              │
│       │    ├── iterations.jsonl                                      │
│       │    ├── session-*.json                                        │
│       │    └── iter-N-summary.md                                     │
│       └── another-project/       ← logs for another project          │
│                                                                      │
│  ~/Documents/yui/sequential-thinking/    (your project — untouched) │
│  ├── GOAL.md          generated by setup.sh                         │
│  ├── score.sh         generated by setup.sh                         │
│  └── ... (your source code)                                          │
└──────────────────────────────────────────────────────────────────────┘
```

hephaestus is installed **once**. You point it at any project via `setup.sh`. The project gets a `GOAL.md` and `score.sh`; the loop runs there and writes logs back here, namespaced by project name.

---

## Installation

```bash
git clone https://github.com/spideynolove/hephaestus ~/tools/hephaestus
cd ~/tools/hephaestus
pip install pyyaml          # only prerequisite
```

## Usage

### First time with a project

```bash
cd ~/tools/hephaestus
bash setup.sh
# → Step 0: enter path to your project, generate GOAL.md + score.sh
# → Step 1: choose LLM provider (OAuth, OpenRouter free, DeepSeek, etc.)
# → Step 2: connection test
# → Step 3: notifications (Slack / Telegram / Discord — optional)
```

### Run the loop

```bash
./orchestrate.sh                 # use .env defaults
./orchestrate.sh --max-iter 5    # cap at 5 iterations
./orchestrate.sh --target 80     # stop at score 80
./orchestrate.sh --dry-run       # preview without executing
```

### Switch to a different project

```bash
bash setup.sh                    # re-run wizard, enter new project path
./orchestrate.sh
```

Each project gets its own log folder under `logs/`. Nothing in your project is touched except `GOAL.md`, `score.sh`, `COMMENTs.md`, and `SUMMARY.md`.

## Setup Wizard (`setup.sh`)

### Step 0 — Project & Goal

Enter the path to the project you want to improve. Then choose how to define the goal:

**Option A — AI-assisted**: `claude` analyzes your project directory and writes `GOAL.md` + `score.sh` automatically based on what it finds.

**Option B — Questionnaire**: answer a few prompts (test runner, linter, coverage tool, objective) and the wizard generates both files from your answers.

Either way, `GOAL.md` follows [goal-md format](https://github.com/jmilinovich/goal-md): fitness function, action catalog, stopping conditions, improvement loop steps, constraints.

### Step 1 — LLM Provider

| Option | Worker | Reviewer | Cost |
|--------|--------|----------|------|
| OAuth subscription | `claude --print` | `claude` | Free |
| OpenRouter free | codex via OpenRouter | claude via OpenRouter | Free (~200 req/day) |
| DeepSeek | codex via DeepSeek | claude via DeepSeek | Paid |
| Z.AI / GLM | `claude --print` | claude via Z.AI | Paid |
| Anthropic + OpenAI | codex native | claude native | Paid |
| Custom endpoint | any OpenAI-compat | any Anthropic-compat | — |

### Step 2 — Connection test

Live call to each configured tool. Shows ✓ or ✗ before writing `.env`.

### Step 3 — Notifications (optional)

Alerts on loop start, improvement, plateau, and completion:
- **Slack** — webhook URL
- **Telegram** — bot token + chat ID
- **Discord** — webhook URL

All tested before saving.

## How the Loop Works

```
repeat until score ≥ target, plateau, or timeout:
  1. Worker reads GOAL.md + COMMENTs.md + MEMORY.md + DECISIONS.md → applies changes → writes SUMMARY.md
  2. score.sh runs in the project dir → outputs 0–100
  3. If improved: git commit [S:NN→NN]; if regressed: git revert
  4. Update MEMORY.md with trajectory, tried actions, observations
  5. Reviewer reads SUMMARY.md + GOAL.md + MEMORY.md + DECISIONS.md → writes new COMMENTs.md
```

### Memory Mechanism

When agent context fills up and sessions restart, three memory files ensure workflow continuity:

| File | Purpose | Max Size |
|------|---------|----------|
| `MEMORY.md` | Workflow state: score trajectory, tried actions, next priorities, observations | 80 lines |
| `STATE.sh` | Machine-readable state for loop restart (scores, iteration count) | ~20 lines |
| `DECISIONS.md` | Immutable directives preventing agent drift | ~15 lines |

These are auto-generated on first run and updated each iteration. Injected into every Worker and Reviewer prompt before GOAL.md.

**Condensation mode**: If the Worker produces empty summaries for N consecutive iterations (configurable), prompts switch to a 3-line condensed format — giving agents maximum context room for small-context providers like DeepSeek.

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Score ≥ target |
| 1 | Plateau — N iterations with no improvement |
| 2 | Max iterations reached |
| 3 | Tests broken after change (immediate revert) |

## Logs

All logs go to `logs/{project-name}/` inside hephaestus — never inside your project.

```
logs/
└── sequential-thinking/
    ├── session-20260408T062346Z.json   ← metadata for one run
    ├── session-latest.json             ← symlink to most recent
    ├── iterations.jsonl                ← one entry per iteration
    ├── iter-1-summary.md               ← Worker's full report, iter 1
    └── iter-2-summary.md
```

**`session-*.json`**
```json
{
  "session_ts": "20260408T062346Z",
  "project": "sequential-thinking",
  "project_dir": "/home/user/Documents/yui/sequential-thinking",
  "worker_tool": "codex",
  "target_score": 95
}
```

**`iterations.jsonl`** — one line per iteration:
```json
{"iteration":3,"before":55,"after":68,"result":"kept","scores":{"score":68,"tests":28,"lint":20,"coverage":20},"action":"Added tests for session_manager...","session":"20260408T062346Z"}
```

**`iter-N-summary.md`** — full Worker progress report, saved after each iteration.

## Scoring

`score.sh` (generated by `setup.sh`, lives in your project):

```bash
bash score.sh           # → 68
bash score.sh --json    # → {"score":68,"tests":28,"lint":20,"coverage":20}
```

Default weights (edit `config.yaml`):

| Component | Max | Default check |
|-----------|-----|---------------|
| Tests | 40 | `npm test` or `pytest` |
| Lint | 30 | `eslint` or `ruff` |
| Coverage | 30 | `jest --coverage` or `pytest-cov` |

`score.sh` is yours to edit. Contract: stdout = single integer; `--json` = JSON with `"score"` key.

## Provider Configuration Reference

Set in `.env` (or via `setup.sh`):

| Variable | Purpose |
|----------|---------|
| `PROJECT_PATH` | Path to the project being improved |
| `ANTHROPIC_API_KEY` | Official Anthropic key |
| `ANTHROPIC_BASE_URL` | Override Anthropic endpoint |
| `ANTHROPIC_AUTH_TOKEN` | Bearer token for custom endpoint |
| `OPENAI_API_KEY` | OpenAI key (used by codex) |
| `OPENAI_BASE_URL` | Override OpenAI endpoint |
| `WORKER_TOOL_OVERRIDE` | Replace codex with claude as Worker |
| `WORKER_MODEL` | Model override for Worker |
| `REVIEWER_MODEL` | Model override for Reviewer |
| `SLACK_WEBHOOK_URL` | Slack notifications |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token |
| `TELEGRAM_CHAT_ID` | Telegram chat/group ID |
| `DISCORD_WEBHOOK_URL` | Discord webhook |

## Prerequisites

- `python3` + `pyyaml` (`pip install pyyaml`)
- `codex` on PATH — or set `WORKER_TOOL_OVERRIDE=claude` to use OAuth
- `claude` on PATH
- `curl` — for notifications
