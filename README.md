# hephaestus

## Prerequisites

- `git`
- `python3` + `pyyaml` (`pip install pyyaml`)
- One of: `codex`, `claude`, or any OpenAI/Anthropic-compatible CLI

## Install

```bash
git clone https://github.com/spideynolove/hephaestus ~/tools/hephaestus
cd ~/tools/hephaestus
bash setup.sh
```

To update:

```bash
git -C ~/tools/hephaestus pull
```

## Usage

Point the loop at a project:

```bash
bash ~/tools/hephaestus/setup.sh
```

Run the loop:

```bash
bash ~/tools/hephaestus/orchestrate.sh
bash ~/tools/hephaestus/orchestrate.sh --max-iter 5
bash ~/tools/hephaestus/orchestrate.sh --target 80
bash ~/tools/hephaestus/orchestrate.sh --dry-run
```

To generate `GOAL.md` + `score.sh` for a project independently:

```bash
bash ~/tools/hephaestus/goal-init.sh /path/to/your/project
```

## Setup Wizard (`setup.sh`)

### Step 0 вҖ” Project

Enter the path to the project you want to improve. If `GOAL.md` + `score.sh` already exist you can keep or regenerate them.

### Step 1 вҖ” LLM Provider (fuel)

| Option | Key needed | Notes |
|--------|------------|-------|
| **OpenRouter** (recommended) | `OPENROUTER_API_KEY` | Any model, one key |
| Anthropic direct | `ANTHROPIC_API_KEY` | Official API |
| Custom OpenAI-compatible | any key | DeepSeek, Ollama, etc. |
| OAuth only | none | claude-code subscription |

Keys saved in `~/.secrets` are loaded automatically.

### Step 2 вҖ” API connection test

Live call to the configured endpoint. On failure: re-enter key / continue / abort.

### Step 3 вҖ” GOAL.md + score.sh generation

Paste context about your goal (type `---` when done). The LLM analyzes your project and generates:
- `GOAL.md` вҖ” objective, fitness function, action catalog, stopping conditions
- `score.sh` вҖ” runnable script using your project's actual test/lint/coverage tools

Falls back to an interactive questionnaire if no API is configured.

### Step 4 вҖ” Execution engine

| Engine | Command |
|--------|---------|
| `codex` | `codex exec --full-auto` |
| `claude` | `claude --print` |
| shell function | `deepseek`, `glm`, `qwen`, `gemini`, etc. |

Worker and Reviewer can use different engines.

### Step 5 вҖ” Notifications (optional)

Slack, Telegram, Discord вҖ” alerts on loop start, improvement, plateau, and completion.

## How the Loop Works

```
repeat until score вүҘ target, plateau, or timeout:
  1. Worker reads GOAL.md + COMMENTs.md + MEMORY.md вҶ’ applies changes вҶ’ writes SUMMARY.md
  2. score.sh runs вҶ’ outputs 0вҖ“100
  3. If improved: git commit; if regressed: git revert
  4. MEMORY.md updated with trajectory, tried actions, observations
  5. Reviewer reads SUMMARY.md + GOAL.md + MEMORY.md вҶ’ writes new COMMENTs.md
```

### Memory files

| File | Purpose |
|------|---------|
| `MEMORY.md` | Score trajectory, tried actions, next priorities |
| `STATE.sh` | Machine-readable state for loop restart |
| `DECISIONS.md` | Immutable directives preventing agent drift |

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Score вүҘ target |
| 1 | Plateau вҖ” N iterations with no improvement |
| 2 | Max iterations reached |
| 3 | Tests broken after change |

## Scoring

`score.sh` lives in your project:

```bash
bash score.sh           # вҶ’ 68
bash score.sh --json    # вҶ’ {"score":68,"tests":28,"lint":20,"coverage":20}
```

Default weights (edit `config.yaml`):

| Component | Max | Default check |
|-----------|-----|---------------|
| Tests | 40 | `pytest` or `npm test` |
| Lint | 30 | `ruff` or `eslint` |
| Coverage | 30 | `pytest-cov` or `jest --coverage` |

`score.sh` is yours to edit. Contract: stdout = single integer; `--json` = JSON with `"score"` key.
