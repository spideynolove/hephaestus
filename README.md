# hephaestus

Automated code improvement loop. Two AI agents alternate — a **Worker** that applies changes and a **Reviewer** that writes targeted feedback — until a fitness score reaches the target or the loop plateaus.

```
┌──────────────────────────────────────────────────────────────────────┐
│  hephaestus/          (lives once, anywhere on your machine)         │
│  ├── orchestrate.sh   loop runner                                    │
│  ├── setup.sh         wizard — points loop at a project              │
│  ├── config.yaml      loop control + agent config                    │
│  ├── .env             active provider + PROJECT_PATH                 │
│  └── logs/                                                           │
│       ├── my-project/      ← logs for one project                    │
│       │    ├── iterations.jsonl                                      │
│       │    ├── session-*.json                                        │
│       │    └── iter-N-summary.md                                     │
│       └── another-project/ ← logs for another project                │
│                                                                      │
│  ~/Documents/my-project/          (your project — untouched)        │
│  ├── GOAL.md          generated by setup.sh                         │
│  ├── score.sh         generated by setup.sh                         │
│  └── ... (your source code)                                          │
└──────────────────────────────────────────────────────────────────────┘
```

hephaestus is installed **once**. Point it at any project via `setup.sh`. The project gets a `GOAL.md` and `score.sh`; the loop runs there and writes logs back here, namespaced by project name.

---

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/spideynolove/hephaestus/main/install.sh | bash
```

That's it. The installer:
1. Clones hephaestus to `~/tools/hephaestus` (override with `HEPHAESTUS_DIR=...`)
2. Checks `python3` and installs `pyyaml` if needed
3. Creates a `heph` command in `~/.local/bin`
4. Launches the setup wizard

To update an existing install, run the same command again — it pulls latest and re-runs setup.

---

## Usage

After install, the setup wizard runs automatically. On future projects:

```bash
bash ~/tools/hephaestus/setup.sh    # point loop at a new project
```

Then run the loop:

```bash
heph                      # run with .env defaults
heph --max-iter 5         # cap at 5 iterations
heph --target 80          # stop at score 80
heph --dry-run            # preview without executing
```

Or directly:

```bash
bash ~/tools/hephaestus/orchestrate.sh
```

## Setup Wizard (`setup.sh`)

### Step 0 — Project

Enter the path to the project you want to improve. If `GOAL.md` already exists you can keep or regenerate it.

### Step 1 — LLM Provider (fuel)

The LLM service provides inference. Supported out of the box:

| Option | Key needed | Notes |
|--------|------------|-------|
| **OpenRouter** (recommended) | `OPENROUTER_API_KEY` | Any model, one key |
| Anthropic direct | `ANTHROPIC_API_KEY` | Official API |
| Custom OpenAI-compatible | any key | DeepSeek, Z.AI, local Ollama, etc. |
| OAuth only | none | claude-code subscription, no generation |

If `OPENROUTER_API_KEY` is saved in `~/.secrets`, it is loaded automatically — no prompt.  
New keys are offered to be saved to `~/.secrets` after a successful test.

### Step 2 — API connection test

Live HTTP call to the configured endpoint. On failure: re-enter key / continue / abort.

### Step 3 — GOAL.md + score.sh generation

Paste context about your goal (type `---` when done, or Enter to skip). The LLM analyzes your project files and generates:
- `GOAL.md` — fitness function, action catalog, stopping conditions
- `score.sh` — runnable script using your project's actual test/lint/coverage tools

Falls back to an interactive questionnaire if no API is configured.

### Step 4 — Execution engine

The engine is the agentic CLI that reads goals and edits code. Separate from the LLM provider:

| Engine | Command |
|--------|---------|
| `codex` | `codex exec --full-auto` |
| `claude` | `claude --print` |
| shell function | `deepseek`, `glm`, `qwen`, `gemini`, etc. |

Worker and Reviewer can use different engines.

### Step 5 — Notifications (optional)

Alerts on loop start, improvement, plateau, and completion — Slack, Telegram, Discord.

## How the Loop Works

```
repeat until score ≥ target, plateau, or timeout:
  1. Worker reads GOAL.md + COMMENTs.md + MEMORY.md → applies changes → writes SUMMARY.md
  2. score.sh runs → outputs 0–100
  3. If improved: git commit; if regressed: git revert
  4. MEMORY.md updated with trajectory, tried actions, observations
  5. Reviewer reads SUMMARY.md + GOAL.md + MEMORY.md → writes new COMMENTs.md
```

### Memory files

| File | Purpose |
|------|---------|
| `MEMORY.md` | Workflow state: score trajectory, tried actions, next priorities |
| `STATE.sh` | Machine-readable state for loop restart (scores, iteration count) |
| `DECISIONS.md` | Immutable directives preventing agent drift |

Auto-generated on first run, updated each iteration, injected into every prompt.

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Score ≥ target |
| 1 | Plateau — N iterations with no improvement |
| 2 | Max iterations reached |
| 3 | Tests broken after change |

## Scoring

`score.sh` lives in your project (generated by setup.sh):

```bash
bash score.sh           # → 68
bash score.sh --json    # → {"score":68,"tests":28,"lint":20,"coverage":20}
```

Default weights (edit `config.yaml`):

| Component | Max | Default check |
|-----------|-----|---------------|
| Tests | 40 | `pytest` or `npm test` |
| Lint | 30 | `ruff` or `eslint` |
| Coverage | 30 | `pytest-cov` or `jest --coverage` |

`score.sh` is yours to edit. Contract: stdout = single integer; `--json` = JSON with `"score"` key.

## `.env` Reference

| Variable | Purpose |
|----------|---------|
| `PROJECT_PATH` | Path to the project being improved |
| `OR_KEY` | API key for the LLM provider |
| `OR_BASE_URL` | LLM API base URL |
| `WORKER_TOOL` | Engine binary/function for Worker |
| `WORKER_FLAGS` | Flags passed to Worker engine |
| `REVIEWER_TOOL` | Engine binary/function for Reviewer |
| `REVIEWER_FLAGS` | Flags passed to Reviewer engine |
| `WORKER_MODEL` | Model name for Worker |
| `REVIEWER_MODEL` | Model name for Reviewer |
| `SLACK_WEBHOOK_URL` | Slack notifications |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token |
| `TELEGRAM_CHAT_ID` | Telegram chat/group ID |
| `DISCORD_WEBHOOK_URL` | Discord webhook |

## Prerequisites

- `python3` (3.8+)
- `git`
- `curl` — for notifications and API calls
- One of: `codex`, `claude`, or any OpenAI/Anthropic-compatible CLI

The installer handles `pyyaml`. Everything else is standard Unix.
